Initial EDA Data Cleansing:

 

Removed Source because it just has 3 values MapQuest, MapQuest-Bing and Bing. This field represents the source of the accident report and its not relevant to predicting accidents.

Removed TMC because it represents how the accident data was collected which is not relevant to causing an accident.

Split Start_Time into 2 columns, Start_Date which represents the date of the accident in MM/DD/YYYY format and Start_Time which represents the time of the accident in hh:mm:ss format.

Split End_Time into 2 columns, End_Date which represents the date of the accident in MM/DD/YYYY format and End_Time which represents the time of the accident in hh:mm:ss format.

Removed End_Lat because 99% of this column has null/blank values.

Removed End_Lng because 99% of this column has null/blank values.

Removed Description because its an open text field which provides details of the accident but from a modeling perspective its not relevant.

Removed Number which represents the street number because of missing values.

Removed Street which represents the street address because we are not interested in analyzing at the street level, our focus is at the city/State/Country level.

Transformed Zipcode to 5 digits to avoid data inconsistency.

Transformed Airport_Code to 3 letters instead of 4 to avoid data inconsistency and mismatch.

Removed Weather_Timestamp because the time the weather was reported has no significant impact on causing an accident.

Removed Wind_Direction because the values in this column are not consistent.

Transformed Wind_Speed(mph) to where if the value is blank replaced it with 0 assuming there was no wind at the time of the accident.

Transformed Precipitation(in) to where if the value is blank replaced it with 0 assuming there was no rain at the time of the accident.

Renamed Amenity to PointofInterest and Give_Way to Yield to avoid confusion.
